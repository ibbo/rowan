Privacy

This page covers the web UI, authentication, chat history, abuse controls, and feedback collection for this ChatSCD deployment.

What we store

  • Chat messages, session IDs, and timestamps so conversations can persist between page reloads.
  • Account details from the sign-in provider you choose, such as name, email, avatar, and login timestamps.
  • Hashed anonymous usage fingerprints, abuse events, request telemetry, and answer feedback to protect the service and review quality.

How we use it

  • Run the assistant and lesson planner, keep chat history available, and connect sessions to signed-in users.
  • Enforce anonymous usage limits, investigate abuse, and monitor reliability and cost.
  • Review feedback and donation-support engagement to improve the service.

Retention and deletion

Chat and operational records are stored in this deployment's SQLite database until they are deleted by the operator or removed through app controls. Use Clear Chat in the main UI to delete the current chat session. For account-linked data or broader deletion requests, use the support contact below.

Third parties

Signed-in access uses configured OAuth providers. Model requests are sent to the language-model provider configured for this deployment. Dance reference data is sourced from The Strathspey Server Dance Database under its published license.
